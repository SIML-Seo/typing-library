# UX 플로우 / 화면 목록 (MVP)

버전: v0.1 (Draft)  
작성일: 2026-03-05

## 0) 원칙

- 핵심: **작품 선택 → 문장 그대로 타이핑(필사) → 결과 확인/저장**
- 오타 정책(합의): **오타가 나도 계속 입력 허용**, 오타 위치 즉시 강조(빨간색), 페이지 종료 시 해당 페이지 오타 요약 리포트
- 타이핑 UI(벤치마크 반영): 원문은 “고스트 텍스트(회색)”로 보이고, 타이핑은 같은 위치에 덮어씌워지는(overlay) 느낌으로 렌더링
- 비용 정책: 유저 결과/내 작품은 로컬 저장(IndexedDB 우선)

## 1) 역할별 화면 접근

- 게스트
  - 공개 작품 탐색/선택
  - 필사/결과/내 기록(로컬)
- 관리자(콘텐츠)
  - MVP에서는 앱 내부 화면이 아니라 “정적 에셋/인덱스 갱신” 운영 프로세스로 공개 작품을 관리

## 2) 화면(IA) 제안 (MVP)

> 라우트는 제안이며, 실제 구현과 1:1로 맞출 필요는 없다.

1. 홈/라이브러리: 공개 작품 목록 + 검색 + 시작 진입점
2. 작품 상세: 작품 메타데이터 + 시작 버튼
3. 이어하기 확인: 저장된 진행 중 세션이 있을 때 이어하기/새로 시작
4. 필사 화면: 타이핑/피드백/지표/페이지 전환
5. 페이지 리포트(모달/시트): 페이지 오타 요약 + “다음 페이지”
6. 완료 리포트: 전체 결과 요약 + 기록 저장 확인
7. 내 기록: 로컬 히스토리(작품/날짜 기준)
8. 내 작품: 텍스트 업로드/목록/삭제(로컬)
9. (레거시/운영) 공개 작품 관리: 정적 에셋/인덱스 갱신(앱 내 관리자 화면은 MVP 제외)

## 3) 핵심 플로우

### 3.1 게스트 — 공개 작품 필사

1) 홈/라이브러리에서 작품을 고른다  
2) 저장된 진행 중 세션이 있으면 “이어하기 / 새로 시작”을 선택한다  
3) 필사 화면에서 문단(=페이지/Chunk) 단위로 입력한다  
4) 입력 중 진행 상태는 로컬에 자동 저장된다  
5) 문단이 끝나면 문단 오타 리포트를 확인하고 다음 문단으로 넘어간다  
6) 작품 완료 시 전체 결과를 확인하고 로컬에 저장되며, 진행 중 드래프트는 제거된다

### 3.2 게스트 — 내 작품 업로드 후 필사(로컬)

1) 내 작품 화면에서 텍스트(파일/붙여넣기)를 업로드한다  
2) 업로드된 내 작품을 선택해 필사한다  
3) 진행 상태는 로컬에 자동 저장되며, 재방문 시 이어하기가 가능하다  
4) 결과/기록은 로컬에 저장된다(동기화 없음)

### 3.3 관리자(운영) — 공개 작품 업로드/관리

1) 저작권 만료 작품 텍스트를 준비한다(출처/근거 포함)  
2) works repo(서브모듈)의 정적 디렉터리에 원문/메타데이터를 추가하고 인덱스를 갱신한다  
3) works 호스팅만 배포한다(예: `firebase deploy --only hosting:works`) → 라이브러리에 노출된다

## 4) 필사 화면 동작 정의(초안)

### 4.1 페이지(Chunk) 개념

- 페이지는 “필사 구간의 단위”다.
- MVP에서는 **문단 기준**으로 정의한다.
  - 문단 분리 규칙(초안)
    - 줄바꿈 정규화: `\r\n` → `\n`
    - 기본: 빈 줄(연속된 `\n\n+`)을 문단 경계로 본다
    - 예외: 텍스트에 빈 줄이 전혀 없으면, 단일 `\n`을 문단 경계로 간주한다(데이터 포맷에 맞춰 조정 가능)

### 4.2 입력/피드백

- 입력은 위치 기반으로 평가한다.
  - `typed[i]`와 `target[i]`를 비교해 해당 위치의 정오표시를 한다.
  - 오타여도 다음 위치 입력을 계속 허용한다.
- 판정 규칙(기본)
  - 공백(space)과 줄바꿈(newline)은 원문과 **완전 일치**로 판정한다.
  - 구두점/대문자도 판정한다(옵션 기본값: **ON**).
- 입력 소스 정책
  - 한글 IME 조합 입력과 일반 키보드 입력은 허용한다.
  - 필사 화면에서 붙여넣기, 드래그앤드롭, 매크로/스크립트성 다문자 입력은 차단한다.
- 렌더링(벤치마크 반영)
  - 목표 텍스트는 고스트(회색)로 먼저 표시
  - 입력이 발생하면 같은 위치에 “입력값”이 덮어씌워지는 느낌으로 표시(정타: 기본색, 오타: 빨간색)
- 오타 표시는 즉시(예: 빨간색).
- Backspace는 허용한다.
  - Backspace로 수정되어 더 이상 불일치가 남지 않으면 오타로 집계하지 않는다.
  - 커서는 일반적인 타자연습 UX(임의 이동 없이 현재 위치에서만 입력).

### 4.3 문단(페이지) 종료 리포트

- 페이지의 마지막 글자까지 입력이 끝나면(또는 “다음 페이지” 액션):
  - 오타 개수
  - 오타가 난 위치/구간(다시 보기 가능한 형태)
  - 다음 페이지로 이동 버튼

### 4.4 결과(완료 리포트)

- 작품 전체 완료 시:
  - 정확도, WPM, 소요시간
  - (선택) 페이지별 오타 통계
- 결과는 로컬에 저장되며, 내 기록 화면에서 조회 가능

### 4.5 자동 저장/이어하기

- 입력 중 세션 드래프트를 로컬에 자동 저장한다.
- 저장된 드래프트가 있으면 작품 진입 시 “이어하기 / 새로 시작” UI를 보여준다.
- 이어하기를 선택하면 마지막 저장 시점의 문단/입력값/누적 시간/설정으로 복원한다.
- 새로 시작을 선택하면 기존 드래프트를 삭제하고 처음부터 시작한다.

### 4.6 설정 패널(MVP 포함)

> 레퍼런스(typelit, typing.works)에서 “가독성/판정/오타 표시” 옵션이 UX 만족도에 큰 영향을 주는 것으로 보인다.

- 가독성: 폰트 크기, 테마(다크 모드), 가독성 폰트
- 판정: 구두점/대문자 포함 여부(특히 외국문학, 기본값: ON)
- 오타 표시 방식: 색상/밑줄 등(기본은 빨간색 강조)

## 5) 엣지 케이스(체크리스트)

- 새로고침/탭 종료 후 이어하기 복원 시점의 시간 계산(비활성 시간은 소요시간에 포함하지 않음)
- 매우 긴 텍스트의 페이지 분절/로딩(공개 작품은 works origin, 내 작품은 IndexedDB)
- 모바일 키보드/한글 조합 입력(IME) 처리
- 붙여넣기/드래그앤드롭/다문자 자동입력 차단이 정상 동작하는지
