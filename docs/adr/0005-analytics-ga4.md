# ADR 0005: DAU 측정은 GA4 사용

상태: Accepted  
작성일: 2026-03-05  
관련 문서: `/docs/prd.md`, `/docs/requirements.md`, `/docs/env.md`, `/docs/deploy.md`

## 컨텍스트

- MVP 성공지표는 DAU(일일 활성 사용자)다.
- 서버가 거의 없는(로컬 저장 중심) 구조에서는 서버 로그 기반 DAU 산정이 어렵다.
- “0원 보장” 제약과 함께 사용할 수 있는 무료 분석 도구가 필요하다.

## 결정

- GA4를 사용해 DAU를 추적한다.
  - (MVP 정의) DAU는 “필사 시작” 이벤트(`typing_start`)를 발생시킨 **유니크 사용자 수(일별)** 로 본다.
    - 랜딩만 보고 이탈한 방문은 성공지표(DAU)에 포함하지 않는다.
- 분석 이벤트에는 아래를 포함하지 않는다.
  - 필사 텍스트(콘텐츠)
  - 토큰/시크릿
  - 이메일 등 직접 식별 가능한 개인정보(PII)
- `NEXT_PUBLIC_GA4_MEASUREMENT_ID`가 있을 때만 GA 태그를 로드하는 방식으로 제어한다(로컬 개발/프라이버시 고려).

관련 결정
- 이벤트 스키마: ADR 0011

## 대안

- 대안 A) 로컬 집계(브라우저에만)
  - 장점: 외부 전송 0, 구현 단순
  - 단점: DAU를 객관적으로 확인 불가(디바이스/브라우저별 중복)
- 대안 B) 서버 로그 기반 집계
  - 장점: 통제 가능
  - 단점: 서버 운영/비용/개인정보 이슈 증가
- 대안 C) GA4(채택)
  - 장점: 무료, DAU 확인 용이
  - 단점: 외부 전송, 쿠키/동의 등 정책 고려 필요

## 결과(Consequences)

- 레이아웃에 GA4 스크립트 삽입 및 핵심 이벤트 정의가 필요하다.
- 프라이버시 정책/동의(필요 시)와 함께 운영해야 한다.
