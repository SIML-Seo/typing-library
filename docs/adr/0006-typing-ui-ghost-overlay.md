# ADR 0006: 타이핑 화면은 “고스트 텍스트 + 오버레이” 렌더링

상태: Accepted  
작성일: 2026-03-05  
관련 문서: `/docs/ux-flows.md`, `/docs/requirements.md`, `/docs/benchmarks.md`

## 컨텍스트

- 레퍼런스(typing.works, typelit)에서 공통적으로 “원문을 바탕(고스트)으로 깔고, 입력이 같은 위치에 덮이는” UX가 확인된다.
- 이 방식은 사용자가 원문을 시선 이동 없이 따라 치기 쉬우며, 학습 흐름이 끊기지 않는다.

## 결정

- 필사 화면은 아래 방식으로 렌더링한다.
  - 목표 텍스트는 고스트(회색)로 표시한다.
  - 사용자가 입력하면 같은 위치에 입력이 덮어씌워지는 느낌으로 표시한다.
  - 정타는 기본색, 오타는 빨간색으로 표시한다.
  - 오타가 나도 경고/모달 없이 계속 진행한다(ADR 0001).

## 대안

- 대안 A) 별도의 입력창(textarea)에 입력하고, 아래/옆에서 정오표시
  - 장점: 구현 단순
  - 단점: 시선 이동 증가, “필사” 느낌이 약함
- 대안 B) 고스트 + 오버레이(채택)
  - 장점: 필사 몰입감, 시선 고정, 레퍼런스 정합
  - 단점: IME/커서/줄바꿈 정렬 등 구현 난이도 상승

## 결과(Consequences)

- 한글 IME 조합 입력에서도 정합한 렌더링/판정이 필요하다.
- 폰트 크기/테마 변경(설정 패널)과 함께 레이아웃/커서 정렬이 깨지지 않도록 해야 한다.
- 오타가 난 위치/구간을 문단 종료 리포트로 연결하는 데이터 구조가 필요하다.
- 공백/줄바꿈까지 원문과 완전 일치로 판정하므로, 사용자가 “공백/줄바꿈 위치”를 인지할 수 있는 보조 UI(예: 다중 공백 가시화)가 필요할 수 있다.
