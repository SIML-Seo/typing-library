# ADR 0003: 사용자 데이터(내 작품/결과)는 로컬 저장만

상태: Accepted  
작성일: 2026-03-05  
관련 문서: `/docs/prd.md`, `/docs/requirements.md`, `/docs/architecture.md`, `/docs/benchmarks.md`

## 컨텍스트

- MVP는 “무료 운영”이 핵심 제약이다.
- 사용자 업로드(내 작품)와 결과(히스토리)를 서버에 저장하면, 스토리지/전송/인증 등 운영 비용이 즉시 발생한다.
- 또한 사용자 텍스트는 개인 자료일 수 있으므로, 서버 전송을 기본값으로 두지 않는 것이 프라이버시에 유리하다.

## 결정

- MVP에서 아래 데이터는 **브라우저 로컬 저장(IndexedDB 우선)** 만 한다.
  - 내 작품(사용자 업로드 텍스트)
  - 결과(필사 히스토리)
- 서버로 업로드/동기화/공유 기능은 MVP에서 제외한다.

## 대안

- 대안 A) 로그인 사용자에 한해 서버 동기화 제공
  - 장점: 기기 변경에도 데이터 유지
  - 단점: 운영 비용 증가, 개인정보/보안 책임 증가, 구현 복잡도 증가
- 대안 B) 전부 로컬 저장(채택)
  - 장점: 비용/보안/프라이버시 측면에서 안전, 구현 단순
  - 단점: 브라우저 초기화/기기 변경 시 데이터 유실

## 결과(Consequences)

- “내 작품/내 기록” UX에서 데이터 유실 가능성을 명확히 안내해야 한다.
- 추후 동기화를 도입할 때는 비용 가드레일과 데이터 정책(암호화/보관기간)을 함께 설계해야 한다.
