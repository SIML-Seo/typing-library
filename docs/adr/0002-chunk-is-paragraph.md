# ADR 0002: 페이지(Chunk) 단위는 문단

상태: Accepted  
작성일: 2026-03-05  
관련 문서: `/docs/ux-flows.md`, `/docs/requirements.md`

## 컨텍스트

- 오타 정책(ADR 0001)을 “페이지 종료 리포트”로 합의하면서, 페이지(Chunk) 정의가 필요해졌다.
- 구현/사용성 관점에서, “문단 단위”가 자연스럽고 저작권 만료 문학 텍스트 포맷과도 잘 맞는다고 판단했다.

## 결정

- MVP에서 페이지(Chunk) 단위는 **문단**으로 한다.
- 문단 분리 규칙(초안)은 아래와 같다.
  - 줄바꿈 정규화: `\r\n` → `\n`
  - 기본: 빈 줄(연속된 `\n\n+`)을 문단 경계로 본다
  - 예외: 텍스트에 빈 줄이 전혀 없으면, 단일 `\n`을 문단 경계로 간주한다

주의
- “문단 경계”로 사용된 줄바꿈은 Chunk 텍스트에 포함되지 않는다(즉, 해당 경계의 Enter 입력은 요구하지 않는다).
- Chunk 내부에 남아있는 줄바꿈 문자는 그대로 유지되며, 타이핑 판정 대상이다(해당 위치에서 Enter가 필요).

## 대안

- 대안 A) 글자 수 기준(예: 700~1200자)
  - 장점: 페이지 길이 균일, UI 예측 가능
  - 단점: 문장/문단이 잘릴 수 있어 필사 경험이 어색해질 수 있음
- 대안 B) 문장 N개 기준
  - 장점: 리듬감
  - 단점: 문장 경계 인식(특히 줄바꿈/따옴표/약어 등)이 복잡해질 수 있음
- 대안 C) 문단 기준(채택)
  - 장점: 의미 단위가 자연스럽고 텍스트 포맷에 가깝다
  - 단점: 원문 포맷이 “줄바꿈만 있고 빈 줄이 없는” 경우 문단 인식이 애매할 수 있음

## 결과(Consequences)

- 타이핑 화면은 문단 단위로 “현재 입력 구간”을 표시한다.
- 문단 종료 시 오타 요약 리포트를 노출한다.
- 실제 작품 데이터 포맷에 따라 문단 분리 규칙을 조정할 수 있다(후속 ADR 또는 문서 업데이트).
