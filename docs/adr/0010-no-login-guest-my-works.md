# ADR 0010: MVP는 로그인 없이 “내 작품” 업로드를 허용

상태: Accepted  
작성일: 2026-03-05  
관련 문서: `/docs/prd.md`, `/docs/requirements.md`, `/docs/auth.md`, `/docs/ux-flows.md`, `/docs/adr/0003-user-content-local-only.md`

## 컨텍스트

- MVP의 성공지표는 DAU이며, 초기 유입/리텐션을 위해 사용 시작 마찰을 최소화해야 한다.
- MVP는 **운영비 0원 보장** 제약이 있어 서버 기능(인증/DB/스토리지/SSR)을 도입하지 않는다(ADR 0004).
- “내 작품/결과”는 로컬 저장만 하므로(ADR 0003), 계정이 없어도 기능이 성립한다.
- 로그인/회원가입을 도입하면 계정 관리, 보안/프라이버시 책임, 구현 복잡도(그리고 비용 리스크)가 증가한다.

## 결정

- MVP는 로그인/회원가입을 제공하지 않는다.
- 대신 “내 작품 업로드/관리”는 로그인 없이 **전 사용자(게스트)** 에게 제공한다.
- 내 작품/결과는 로컬(IndexedDB) 저장만 하며, 서버 업로드/동기화는 MVP에서 제외한다.

## 대안

- 대안 A) “회원만 내 작품 업로드” (로그인 게이트)
  - 장점: 정책적으로 업로드를 제한할 수 있음
  - 단점: 회원가입 마찰 증가, 인증 도입 비용/리스크 증가
- 대안 B) 로그인은 선택(게스트 + 선택적 계정)
  - 장점: 일부 사용자에게만 동기화 제공 가능
  - 단점: 결국 서버/계정/동기화가 필요해 MVP 범위를 넘어설 수 있음
- 대안 C) 로그인 없이 전 사용자에게 로컬 업로드 제공(채택)
  - 장점: 마찰 최소, 비용/보안 책임 최소, 구현 단순
  - 단점: 기기 변경/브라우저 초기화 시 데이터 유실

## 결과(Consequences)

- “내 작품/내 기록” 화면에 **데이터 유실 가능성**(브라우저 삭제/기기 변경)을 명확히 안내해야 한다.
- 추후 동기화/공유/계정이 필요해지면, 0원 보장과 충돌하지 않는 방식으로 별도 ADR로 재결정한다.
