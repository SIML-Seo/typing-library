# 레퍼런스 / 벤치마크 정리

버전: v0.1 (Draft)  
작성일: 2026-03-05

## 0) 목적

외부 타이핑/필사 서비스에서 “타이핑 UX/옵션/지표 표기” 패턴을 참고해, 우리 문서(PRD/요구사항/UX 플로우)를 구체화한다.

## 1) Typelit

링크: https://www.typelit.io/

관찰/정보(공개 페이지·리뷰 + 화면 관찰)
- 구두점(punctuation)·대문자(capitalization) 판정을 끄는 옵션이 언급됨
- 폰트/테마(커스텀) 등 가독성 옵션이 존재(예: OpenDyslexic 언급)
- (Premium) EPUB/PDF/TXT import, 커스텀 텍스트 붙여넣기/작성 기능이 있음
- import한 텍스트는 “서버가 아니라 브라우저에만 저장”한다는 정책이 명시됨
- 타이핑 화면에서 목표 텍스트를 바탕(고스트)로 깔고, 타이핑 결과가 같은 위치에 덮여 보이는 패턴
- 오타는 즉시 빨간색으로 표시되며, 경고/모달 없이 계속 입력 가능
- 상단에 WPM/ACC(정확도) 같은 지표가 실시간으로 노출되는 UI가 확인됨
- 옵션 기본값(구두점/대문자 판정 on/off, 오타 집계 방식 등)은 설정 패널/도움말 확인이 필요(문서 기반으로 단정하지 않음)

우리에게의 시사점
- **비용/프라이버시**: “내 작품 업로드”는 IndexedDB 로컬 저장으로 시작(서버 업로드 금지) — 이미 합의된 방향을 강화
- **학습 난이도 옵션**: 구두점/대문자 판정 on/off 같은 옵션은 특히 외국문학에서 유용(MVP 포함)
- **가독성/접근성**: 폰트 크기/테마(다크 모드)/가독성 폰트 옵션은 UX 품질에 직접 영향

## 2) typing.works

링크: https://typing.works/

관찰/정보(앱 소개 페이지 + 화면 관찰)
- 즐겨찾기/기록(히스토리) 중심 UX(출석/연속일 등)
- 옵션: 폰트 크기/스타일, 타이핑 사운드(외부 키보드), 오타 표시 방식(typo display method), 다크 모드 등
- 타이핑 화면에서 목표 텍스트(고스트)가 바탕에 있고, 입력이 같은 위치에 덮여 보이는 패턴
- 오타는 즉시 빨간색으로만 표시(알림/경고 없이 진행)
- 옵션 기본값/정확도 집계 방식은 화면 관찰만으로 확정하기 어려움(설정 패널 확인 필요)

우리에게의 시사점
- **설정 패널 필요성**: 최소한 “폰트 크기/테마/오타 표시 방식”은 사용자 만족도를 좌우할 수 있음(MVP 포함)
- **동기부여 요소**: 출석/연속일, 즐겨찾기, 퀵 리스트는 DAU에 도움이 될 수 있으나 MVP에서는 후순위로 두는 것이 안전

## 3) 우리가 당장 반영할 것(문서 수준)

- 페이지(Chunk) 단위: 문단(이미 결정)
- 오타 정책: 진행 허용 + 즉시 강조 + 문단 종료 리포트(이미 결정)
- 판정/편집 정책(고정)
  - 공백/줄바꿈 완전 일치
  - 구두점·대문자 판정 기본 ON(옵션으로 on/off)
  - Backspace 수정 가능(수정되어 불일치가 없어지면 오타로 집계하지 않음)
- 내 작품/결과 저장: 로컬(IndexedDB) 우선(이미 방향 합의)
- 설정 패널(MVP): 폰트 크기/테마/구두점·대문자 판정/오타 표시 방식
