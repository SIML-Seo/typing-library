# 테스트 전략 / E2E 시나리오

버전: v0.1 (Draft)  
작성일: 2026-03-05

## 0) 목표

- 타이핑 UX(오타 정책/페이지 리포트)가 깨지지 않도록 “핵심 로직”을 테스트로 고정한다.
- 배포 전 최소 스모크 테스트로 works 로딩/로컬 저장/설정 패널 등 핵심 기능의 회귀를 막는다.

## 1) 테스트 스택 / 피라미드

- Unit/Integration: `Vitest` + `React Testing Library`
- E2E: `Playwright`
- 목적: 타이핑 엔진의 판정 로직은 Unit으로 고정하고, 핵심 사용자 흐름은 E2E로 검증한다

1) Unit (가장 중요)
- 텍스트 분절(문단/Chunk)
- 입력 처리(한글 IME 포함)
- 입력 소스 제한(붙여넣기/드래그앤드롭/다문자 삽입 차단)
- 오타 계산/표시용 데이터 구조
- 판정 옵션(구두점/대문자 on/off)
- 설정 옵션(폰트/테마/오타 표시) 적용
- WPM/정확도/시간 계산

2) Integration
- IndexedDB 저장/조회(내 작품/결과)
- 공개 작품 로딩(정적 인덱스 + 정적 텍스트 fetch)

3) E2E
- 게스트 필사 플로우
- 문단 종료 오타 리포트
- 결과 저장/히스토리 조회
- 설정 패널(폰트/테마/판정/오타 표시)

## 2) E2E 시나리오(체크리스트)

### E2E-01 게스트: 공개 작품 필사 + 페이지 리포트

- [ ] 작품 목록에서 작품 선택
- [ ] 필사 화면 진입
- [ ] 일부러 오타 입력 → 즉시 빨간색 표시 확인
- [ ] 붙여넣기 시도 → 입력 반영되지 않음 확인
- [ ] 드래그앤드롭/비정상 다문자 입력 시도 → 입력 반영되지 않음 확인
- [ ] 문단(=페이지) 완료 → 페이지 오타 요약 리포트 노출 확인
- [ ] 다음 페이지 진행
- [ ] 작품 완료 → 결과 요약 표시 확인

### E2E-02 게스트: 결과 로컬 저장/조회

- [ ] 필사 완료 후 “내 기록”에서 결과가 보인다
- [ ] 새로고침 후에도 결과가 유지된다(IndexedDB)

### E2E-03 게스트: 진행 상태 자동 저장/이어하기

- [ ] 필사 도중 몇 글자 입력
- [ ] 새로고침 또는 페이지 재진입
- [ ] “이어하기” UI 노출 확인
- [ ] 이어하기 선택 시 마지막 입력 상태/문단/누적 시간 복원 확인
- [ ] 새로 시작 선택 시 기존 드래프트 삭제 후 처음부터 시작 확인

### E2E-04 게스트: 내 작품 업로드(로컬) 후 필사

- [ ] 내 작품 업로드(파일/텍스트)
- [ ] 업로드된 작품으로 필사 가능
- [ ] 결과가 로컬에 저장/조회 가능

### E2E-05 관리자: 공개 작품 관리

MVP는 정적 사이트로 운영하며 앱 내 “관리자 업로드”를 제공하지 않는다.  
공개 작품 추가는 운영 프로세스(정적 에셋/인덱스 갱신)로 처리한다.

### E2E-06 정적 콘텐츠 배포 확인

- [ ] 작품 인덱스(`{worksBaseUrl}/works/index.json` 등)가 로드된다
- [ ] 작품 원문(`{worksBaseUrl}/works/{id}.txt` 등)이 로드된다

### E2E-07 설정 패널(MVP): 폰트/테마/판정/오타 표시

- [ ] 필사 화면에서 설정 패널을 연다
- [ ] 폰트 크기 변경 → 레이아웃/커서/판정이 깨지지 않는다
- [ ] 다크 모드 전환 → 대비가 충분하고 입력/표시가 유지된다
- [ ] 구두점/대문자 판정 on/off → 정확도/오타 계산 기준이 일관되게 바뀐다
- [ ] 오타 표시 방식 변경 → 성능 저하(입력 지연) 없이 스타일만 바뀐다

## 3) 수동 스모크 테스트(배포 전)

- [ ] 게스트 필사 1회 완료
- [ ] 필사 중 새로고침 후 이어하기 복원 확인
- [ ] 페이지 오타 리포트 확인
- [ ] 로컬 기록 유지(새로고침)
- [ ] GA4 태그 로드 및 핵심 이벤트(typing_start/typing_complete) 수신(콘텐츠 미전송, DAU=typing_start)

## 4) 알려진 제약

- 한글 IME 조합 입력은 브라우저/OS 차이가 있으므로, 자동화 테스트와 별도로 수동 스모크 테스트를 유지한다.
- IME 관련 버그가 발생하면 Playwright 재현만으로 닫지 말고 실제 한국어 입력 환경에서 다시 확인한다.
