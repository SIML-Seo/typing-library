# PRD — Typing Library (필사형 타이핑)

버전: v0.1 (Draft)  
작성일: 2026-03-05  
상태: 초안(문서 기반 개발)

## 1) 한 줄 소개

저작권이 만료된 문학 작품을 선택해 “문장을 그대로 타이핑(필사)”하고, 결과를 로컬에 저장해 반복 연습하는 웹앱.

## 2) 목표(Goals)

- 핵심 플로우(작품 선택 → 필사 → 결과)를 빠르고 안정적으로 제공
- 유입을 위해 “랜딩(마케팅) 페이지”의 SEO 기본기를 갖춘다(정적 생성 기반, **검색 노출은 랜딩만**)
- 운영비가 유료 플랜/서버 비용으로 이어지지 않도록 “클라이언트 중심”으로 설계
- 성공지표: DAU(일일 활성 사용자)

## 3) 제약(Constraints)

- 무조건 무료(운영비 0원 보장)
  - 원칙: 유저 증가가 곧바로 “서버 쓰기/저장 비용” 증가로 이어지는 기능은 MVP에서 제외
  - 자동 과금/종량 과금으로 비용이 발생할 수 있는 구성은 MVP에서 지양하며, 필요 시 “하드 캡(초과 시 기능 제한)”으로 비용이 0원을 유지해야 한다
  - MVP(개발/배포)는 **AWS(Amplify/Cognito/AppSync/S3) 등 종량 과금형 인프라를 사용하지 않는다**
  - 배포는 **정적 사이트(Static)** 를 원칙으로 한다(서버 런타임/DB 없이 동작)
  - SEO가 필요하더라도 MVP는 SSR(요청당 서버/엣지 렌더링) 없이, 정적 생성(SSG/정적 export)으로 해결한다
  - 앱 기능 화면(필사/기록 등)은 SEO 대상이 아니므로 색인되지 않도록 설정한다(예: noindex)
  - 유저 데이터(결과/개인 업로드)는 기본 로컬 저장(IndexedDB/LocalStorage)
- 공개 작품은 저작권 만료 작품만 취급(작품별 출처/근거를 메타데이터로 남김)

## 4) 사용자/권한

- 게스트(비로그인)
  - 작품 선택 및 필사 가능
  - 내 작품 업로드/관리 가능(로컬)
  - 결과/기록은 로컬 저장
- 관리자(콘텐츠)
  - 공개 작품 라이브러리 업로드/관리는 정적 에셋/인덱스를 갱신하는 운영 프로세스로 처리한다(앱 내 관리자 페이지는 MVP 범위에서 재검토).

## 5) 핵심 사용자 플로우

1) 작품 탐색/선택  
2) 필사 화면 진입  
3) 문단 표시 → 사용자는 동일하게 타이핑  
4) 진행률/오타 피드백/시간 측정  
5) 완료 시 결과 요약(정확도/WPM/소요시간 등)  
6) 결과 저장(로컬) 및 기록 조회(로컬)

## 6) MVP 기능 범위

### 6.1 작품 라이브러리(공개)

- 작품 목록(제목/저자), 기본 검색/필터(최소)
- 작품 상세 + 시작 버튼
- 텍스트 로딩
  - 공개 작품 텍스트는 **작품 전용 정적 호스팅(works origin)** 에서 제공한다(앱 배포와 독립적으로 업데이트).
    - 예: `GET {worksBaseUrl}/works/index.json`, `GET {worksBaseUrl}/works/{id}.txt`
  - 작품 텍스트가 커질 경우 “정적 분할(파트/챕터) + 캐싱(브라우저 캐시/Service Worker 등)”로 트래픽을 완화한다(서버 없이).

### 6.2 필사(타이핑) 경험

- 표시 단위: **문단(=페이지/Chunk)** 기준
- 타이핑 UI(벤치마크 반영: typing.works / typelit)
  - 목표 텍스트를 “고스트 텍스트(회색)”로 먼저 표시하고, 사용자가 타이핑하면 같은 위치에 입력이 **덮어씌워지는(오버레이)** 느낌으로 렌더링
  - 오타는 즉시 빨간색으로만 표시(경고/모달 없이 계속 진행)
- 입력 판정 규칙(고정)
  - 공백/줄바꿈은 원문과 **완전 일치**로 판정
  - 구두점·대문자도 판정(옵션 기본값 **ON**)
  - 오타 시 동작: **오타가 나도 진행은 계속 허용**, 오타 위치를 즉시 강조(빨간색). 페이지 종료 시 해당 페이지의 오타를 요약 리포트.
  - Backspace로 수정 가능(수정되어 불일치가 없어지면 오타로 집계하지 않음). 커서는 일반적인 타자연습 UX(임의 커서 이동 없이 현재 위치에서만 입력)
- 실시간 지표
  - 진행률, 오타 수, 정확도, WPM, 경과 시간
- 일시정지/재시작
- 진행 상태 자동 저장/이어하기
  - 필사 중인 세션은 로컬에 자동 저장한다
  - 새로고침/탭 종료/브라우저 재방문 후에도 같은 기기에서 “이어하기”를 제공한다
- 설정 패널(MVP 포함)
  - 폰트 크기/테마(다크 모드)
  - 구두점·대문자 판정 on/off(특히 외국문학)
  - 오타 표시 방식

### 6.3 결과 저장(무료 운영 핵심)

- 결과는 기본 로컬 저장(IndexedDB 우선, LocalStorage는 요약치)
- 진행 중 세션 드래프트도 로컬 저장한다(이어하기용)
- 기록 조회: 날짜별/작품별 히스토리(로컬)

### 6.4 내 작품 업로드(비용 방지)

- 로그인 없이 누구나 “내 작품”을 추가/삭제 가능
- 저장은 로컬 IndexedDB (서버 업로드/Storage 업로드는 MVP 제외)
- 내 작품도 동일한 필사/기록 기능에 사용
  - 업로드 방식은 텍스트 파일(`.txt`) 선택 또는 텍스트 붙여넣기를 모두 지원한다
  - 단, 필사 화면에서는 붙여넣기를 허용하지 않는다(업로드 화면과 구분)

### 6.5 관리자 기능

- 공개 작품 업로드(대량) + 메타데이터 관리(제목/저자/장르/출처/근거 등)
- 저작권 만료 여부 체크리스트(운영 프로세스 문서화)

## 7) DAU 측정(분석)

- 목표지표: DAU
- 방식: GA4 사용(외부 무료 웹 분석 도구)
  - (MVP 정의) DAU는 `typing_start` 이벤트를 발생시킨 유니크 사용자 수(일별)로 본다(랜딩 이탈 제외, ADR 0011)
  - 이벤트(최소): `typing_start`, `typing_paragraph_complete`, `typing_complete`, `my_work_upload` (ADR 0011)

## 8) 비목표(Non-goals) — MVP 제외

- 로그인/회원가입/계정 관리
- 랭킹/대결/소셜 공유/친구 기능
- 유료화, 구독, 결제
- 기기간 동기화(서버에 기록 저장)
- AI 코칭/자동 교정

## 9) 후속 운영 작업

- works repo에서 인덱스 생성/검증 스크립트와 배포 절차를 구체화한다
- 런칭 전 `Firebase Hosting (Spark)` 가드레일을 체크리스트 기준으로 최종 점검한다
